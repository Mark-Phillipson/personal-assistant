# Questionnaire response

Date: 2026-05-31

1) About me and daily tech use

I am a freelance web developer who works from home (since 2012) because working in an office reduces my productivity. I use Talon (voice) alongside a Tobii eye tracker for gaze control. I have RSI and can no longer drive without causing pain, so voice and gaze interaction are essential. These tools let me code, navigate, and manage communications with much less physical strain.

2) Assistive technologies I use and how I use them

- Primary tools: Talon (voice) and a Tobii eye tracker (gaze).
- Talon add-ons/extensions: Visual Studio Code Cursorless extension, Rango (browser extension), Talon Heads-Up Display, talon-file-tree, Talon Gaze mouse grid, talon-gaze OCR, and a physical mouse grid.
- AI and dictation stack: I primarily code with GitHub Copilot and often dictate into a small text box so AI can draft and format content for me. For speech->text I use Talon's D2 engine together with Whisper, and I run a local Ollama model to post-correct and refine dictation. I also use a GitHub Copilot custom agent for more complex generation tasks.

a) How long: I've been using Talon, Tobii and related workflows for several years and have added tools progressively.

b) How my approach changed: I initially started out using just a microphone with Microsoft Office's built-in dictation software, mainly to reply to emails and reduce keyboard use. I then moved on to using Dragon NaturallySpeaking full time, before eventually switching to Talon for its programmable command chaining, specialised add-ons for precise editing/navigation.

c) Why these tools: Talon provides programmable, high-precision voice control; Tobii supplies reliable gaze input; local models (Ollama/Whisper) give low-latency, private dictation correction; Copilot speeds up coding and drafting.

d) Contexts they work well: Code editing and navigation, drafting and formatting text via AI, web browsing and many productivity tasks where exact spatial manipulation is not the primary requirement.

I have created several YouTube videos showing me using the assistive tech: https://youtu.be/5c-cBXjn1Ro

Main pain points:
- Dictation: when I dictate slowly I end up with lots of unwanted punctuation because Whisper tends to work best with faster, continuous speech without pauses.
- Meetings: switching between talking to the computer and talking to a person is difficult and interrupts workflow.
- Mouse/gaze: general mouse-like interactions remain a pain point; nothing matches the precision and speed of a physical mouse.
- Cognitive load: remembering the thousands of custom voice commands is hard and can be overwhelming.

History: The main reason I switched from Dragon NaturallySpeaking three years ago was the desire to chain commands together more flexibly. Dragon also appeared to be lagging in updates compared with rapid AI enhancements and the growing ubiquity of speech recognition across other tools and platforms.

4) Complex workflows / specialised contexts

Because I freelance and bid for specific jobs, I naturally gravitated toward text-based work (HTML, CSS, code) rather than drag-and-drop visual design tools. This meant I could largely avoid the spatial interaction challenges that tools like Figma or Illustrator would present. For diagrams I rely on Mermaid inside Visual Studio Code — I can describe a diagram in text, let AI generate the Mermaid syntax, then voice-edit and tweak it. This keeps the entire workflow keyboard/voice-friendly and avoids any drag-and-drop canvas interaction.

5) What I wish could change

When you install new software, it automatically hooks up keyboard and mouse support from the start. I wish the same were true for microphone and speech recognition — that voice control was treated as a first-class input method right from the word go, on equal footing with keyboard and mouse, rather than something you have to retrofit afterwards.
